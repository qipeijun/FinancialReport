# 每日财经分析一键流程

## 重要：分步执行，每步必须向用户汇报

**不要静默跑完再汇报**。每完成一个步骤，立刻向用户汇报该步的关键结果和数字。

**耗时必须精确**：每个步骤用 `time` 命令包裹，取 `real` 值作为实际耗时。**禁止**使用脚本内部打印的耗时（如"耗时 4.7s"），因为那往往只是子阶段的时间，不代表整步的真实 wall clock 时间。

## 执行流程

### 步骤1：抓取RSS新闻

```bash
time ./venv/bin/python scripts/rss_finance_analyzer.py --fetch-content
```

完成后汇报：几源成功/失败、抓取篇数、新增入库篇数、wall clock 耗时。

### 步骤2：AI分析生成报告

```bash
time ./venv/bin/python scripts/ai_analyze_deepseek.py --content-field summary --model deepseek-chat
```

完成后汇报：分析文章数、质量评分Top3标题及分数、Token消耗、wall clock 耗时。

### 步骤3：更新导航 + 构建站点

```bash
time ./venv/bin/python scripts/generate_mkdocs_nav.py && time mkdocs build
```

分别汇报导航生成耗时和站点构建耗时。

### 步骤4：联网查询实时数据

**在提炼摘要前，必须联网搜索以下信息。分批并行搜索，提高效率。**

**第一波（并行搜索）：**

1. 搜索"2026年6月X日 A股 收盘 上证指数 深证成指 创业板"，获取三大指数实际收盘点位和涨跌幅
2. 搜索"2026年6月X日 美股 道琼斯 纳斯达克 收盘"，获取美股前日收盘数据
3. 搜索"布伦特原油 COMEX黄金 价格"，获取商品实时报价
4. 搜索"2026年6月X日 财经 要闻 突发"，获取报告可能未覆盖的突发事件

**第二波（并行搜索）：**

读取报告后，提取报告中推荐的**核心股票**（至少5只），逐个搜索其当日行情和重大消息：
5. 搜索"[股票代码] [股票名称] 2026年6月X日 股价 涨跌"，获取每只标的的实际涨跌幅
6. 对报告中提到的重大事件（如并购、业绩预告、政策），搜索验证具体数据

**查询要求：**
- 用 `WebSearch` 工具
- 如果6月23日当天A股尚未收盘，搜索"6月23日盘中"获取最新数据
- 第一波和第二波可以分开进行（第二波需要先读报告才知道哪些股票）
- 实时数据与报告数据对照，以实时数据为准，有重大出入标注差异

### 步骤5：读取报告并提炼摘要

报告路径：`docs/archive/YYYY-MM/YYYY-MM-DD/reports/📅 YYYY-MM-DD 财经分析报告_morning_deepseek.md`

## 摘要格式要求

摘要必须结合**报告内容 + 联网实时数据**，以实时数据为最终依据。

---

### 📈 大盘表现

| 市场 | 指数 | 点位 | 涨跌 |
|------|------|------|------|
| A股 | 上证 | XXXX | +X.X% |
| A股 | 创业板 | XXXX | -X.X% |
| 美股 | 纳指 | XXXX | -X.X% |
| ... | ... | ... | ... |

一句话概括今日市场情绪。

---

### 🔥 投资主题

对报告中每个重点投资主题，按 serenity 方法论提炼。**每个主题必须覆盖以下 5 个维度：**

**主题：[名称]**

- **产业链卡点在哪里** —— 这个主题背后，供给链中真正紧张、难以扩产的环节是什么？不是泛泛说"算力紧张"，要定位到具体层级（如：HBM 封装产能、光纤预制棒、液冷冷板加工）。
- **发生了什么** —— 2-3 句讲清逻辑。
- **核心标的** —— 3-5 个具体股票/代码，涨跌幅**必须来自联网搜索**，不能用报告里的过时数据。标注每条价格数据的来源（实时/盘中等）。
- **市场可能没看清的地方** —— 有什么被表面涨跌掩盖的结构性变化？（如：光模块大跌不是基本面恶化，而是液冷路线切换导致资金重估）
- **反方理由** —— 什么情况说明这个判断错了？必须给出具体可验证的条件，不是泛泛说"市场风险"。
- **后续催化剂** —— 接下来要盯什么事件/数据。

**证据强度标注**：每条关键判断后面标上证据强度：
- `[强]` — 来自财报/官方公告/交易所文件
- `[中]` — 来自权威财经媒体/券商研报
- `[弱]` — 来自自媒体/社交平台/单源传闻

---

### 🎯 优先研究方向

用 serenity 的"先排产业链层级，再排公司"逻辑：

- **短期最值得研究的层级** — 哪个产业链环节当前卡点最紧、证据最强？
- **中期配置主线** — 3-12 月哪些方向值得持续跟踪？
- **现阶段应该回避的方向** — 哪些是拥挤交易/故事驱动的？

注意：这里给的是**研究优先级**，不是买卖建议。把交易决策留给用户。

---

### ⚠️ 最大风险（反方视角）

挑最重要的 2-3 个风险，每个回答 serenity 的核心问题：**"什么具体可观测的信号出现，就说明这个整体判断是错的？"**

---

### 📅 本周关键事件

事件列表，每项标注市场预期和可能的影响方向。

---

**表达要求**：
- 用中文，像研究伙伴在跟你说话，不要报告腔
- 数据具体到数字，涨跌必须写百分比
- 产业链卡点分析是摘要中最有价值的部分，不要敷衍
- 不添加报告中不存在的信息，除非来自联网查询并标注来源
